# g12

## English

**One-liner.** Vietnam grade-12 standardized-exam platform across 11 subjects (English, Math, Physics,
Chemistry, Biology, Geography, History, Civic, Informatics, Agri/Industrial Tech): test authoring,
student execution, AI explanation generation, course/classroom integration, analysis & ranking.

**Stack.** Baseline + Kafka V2. Polymorphic `G12ExamTest` / `G12UserExamTestProgress` by subject.

**Features.**
- **Test data entry** — `/api/entry/*`: create/update/delete/clone/publish, by-ids, paged by subject.
  Entities `G12ExamTest` (`g12ExamTest`, subject-polymorphic), `Question` (MCQ/TrueFalse/ShortAnswer), `QuestionGroup`, versioned.
- **Sheet import/export** — `/api/entry/file/validate|import`, `GET /public/api/entry/export/{testId}` (Apache POI,
  Excel/Google Sheets). `ImportQueue` tracks in-flight jobs.
- **AI explanation generation** — `/api/hook/*`: webhook receive, queue-status, retry-one/all. External AI queue
  (model `g12-gpt-5.2`, image read), keys `{examId}__{questionKey}__{explanationType}`. Entities `G12AIStatus`, `G12AIResult`.
- **Student test execution** — `/public/api/do-test/*`: check-status, start, save, check-answer, submit, extend-time,
  change-mode. Entity `G12UserExamTestProgress` (`user_exam_test_progress`): answers, score, statistic, mode (PRACTICE/CHALLENGE).
- **Course-integrated execution** — `/api/course-do-test/*` + teacher endpoints; integrates `coursesupporter` (filter/statistic),
  tracks userResourceId / studentRegistrationKey / subClassKey.
- **Analysis & ranking** — `/api/analyze/progress/{id}`, history + leaderboard (`/rank`).
- **Change history** — `G12ExamTestListener` (Mongo BeforeSave) diffs explanations → `ActionHistoryRestClient` → [[action-history]].
- **Public preview / SEO / sync** — preview, `QuestionSeoController`, initial-history sync.

**Data model.** `g12ExamTest`, `user_exam_test_progress`, `g12ExamTestVersion`, `importQueue`, `g12_ai_status`.

**Integrations.** page-management, [[action-history]] (field changes), english-utils, s3-service-v2, editor-utils,
Directus (AI prompts), coursesupporter; Google Sheets, vCDN; Kafka V2 topics `g12-service/receive_user_do_test_action`, `.../completed_test`.

**Gotchas.** Subject polymorphism (JSON discriminator) — QueryDSL must handle it. Tests versioned via
`DocumentVersionKey`; progress pins the version at start. Mode switch PRACTICE→CHALLENGE mid-test. Only explanation changes are tracked.

## Tiếng Việt

**Một dòng.** Nền tảng thi chuẩn lớp 12 cho 11 môn (Anh, Toán, Lý, Hoá, Sinh, Địa, Sử, GDCD, Tin, Công nghệ
NN/CN): soạn đề, học sinh làm bài, AI sinh giải thích, tích hợp khoá học/lớp, phân tích & xếp hạng.

**Stack.** Baseline + Kafka V2. `G12ExamTest` / `G12UserExamTestProgress` đa hình theo môn.

**Chức năng.**
- **Nhập liệu đề** — `/api/entry/*`: tạo/sửa/xoá/clone/publish, theo nhiều id, phân trang theo môn. Entity
  `G12ExamTest` (đa hình theo môn), `Question` (MCQ/TrueFalse/ShortAnswer), có version.
- **Import/export sheet** — `/api/entry/file/validate|import`, export Excel (Apache POI, Google Sheets). `ImportQueue` theo dõi job.
- **AI sinh giải thích** — `/api/hook/*`: nhận webhook, queue-status, retry. Queue AI ngoài (model `g12-gpt-5.2`),
  key `{examId}__{questionKey}__{explanationType}`. Entity `G12AIStatus`, `G12AIResult`.
- **Học sinh làm bài** — `/public/api/do-test/*`: check-status, start, save, check-answer, submit, extend-time,
  change-mode. Entity `G12UserExamTestProgress` (đáp án, điểm, thống kê, mode PRACTICE/CHALLENGE).
- **Làm bài trong khoá học** — `/api/course-do-test/*` + endpoint giáo viên; tích hợp `coursesupporter`,
  theo dõi userResourceId / studentRegistrationKey / subClassKey.
- **Phân tích & xếp hạng** — `/api/analyze/progress/{id}`, lịch sử + bảng xếp hạng (`/rank`).
- **Lịch sử thay đổi** — `G12ExamTestListener` so sánh giải thích → đẩy về [[action-history]].
- **Preview công khai / SEO / sync** — preview, SEO câu hỏi, sync lịch sử ban đầu.

**Data model.** `g12ExamTest`, `user_exam_test_progress`, `g12ExamTestVersion`, `importQueue`, `g12_ai_status`.

**Tích hợp.** page-management, [[action-history]], english-utils, s3-service-v2, editor-utils, Directus, coursesupporter;
Google Sheets, vCDN; Kafka V2 (`g12-service/receive_user_do_test_action`, `.../completed_test`).

**Lưu ý.** Đa hình theo môn (JSON discriminator) — QueryDSL phải xử lý. Đề có version qua `DocumentVersionKey`;
progress ghim version lúc bắt đầu. Đổi mode PRACTICE→CHALLENGE giữa chừng. Chỉ track thay đổi giải thích.
